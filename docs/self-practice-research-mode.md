# Self-Practice Research Mode

This project should now be interpreted primarily as an **AI self-practice intervention system** rather than a teacher-evaluation system.

## Core research focus

The main research question is no longer:

- whether the AI agrees with teacher judgment

The main research question is now:

- whether AI-supported self-practice improves erhu learners' learning outcomes
- whether learners understand and use the structured feedback during independent practice
- whether the tool improves perceived usefulness, feedback clarity, confidence, and continuance intention

## Intervention definition

The intervention is:

- learner selects a piece or section
- learner records or uploads a performance
- the system analyzes pitch and rhythm
- the system highlights problem notes and measures
- the system gives demo playback and structured practice advice
- the learner decides how to revise the next take

This means the intervention is **AI feedback + self-regulated practice**, not teacher-guided correction.

## What remains primary

- whole-piece / section-level analysis
- structured feedback
- demo playback
- repeated self-practice logs
- pretest / posttest comparison
- usage logs
- questionnaires
- interviews

## What becomes optional

The following modules remain available in the project, but they are **optional validation layers**, not the main study axis:

- teacher validation review
- dual-rater reliability
- adjudication workflow
- teacher batch review sheets

Use them only if later you need external validity checks. They are not required for the self-practice study design.

## Recommended main outcome domains

1. Learning effect
   - pitch gain
   - rhythm gain
   - repeated-practice improvement across takes

2. Learning experience
   - perceived usefulness
   - ease of use
   - feedback clarity
   - confidence
   - continuance intention

3. Self-practice behavior
   - number of analyses
   - number of repeated takes
   - whether learners follow the suggested path
   - which feedback components they revisit most

## Recommended next step

Run a small pilot that emphasizes:

- learner self-practice flow
- system log completeness
- questionnaire completion
- interview evidence on how learners interpret the feedback

Do not block this stage on teacher validation unless you explicitly decide to add it as an auxiliary evaluation layer.
